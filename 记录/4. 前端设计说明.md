# WMS Lite 前端设计说明

> 需求明确"不要求实现完整前端页面"，但前端的核心设计思路需要清晰。

---

## 1. 技术选型

| 层面 | 选型 | 理由 |
|------|------|------|
| 框架 | React | 组件化 |
| 样式 | Tailwind CSS | 轻量、不依赖组件库 |
| 路由 | React Router | 前端路由守卫、页面切换 |
| 请求 | Axios / fetch | HTTP 请求，统一拦截器处理 Token 和错误 |
| 状态 | React Context / useState | 全局存用户信息（token、role），页面级用 useState 即可 |

---

## 2. 认证流程（前端视角）

### 2.1 登录

```
用户输入用户名 + 密码
  → POST /api/auth/login
  → 后端返回 { token, role }（role 是明文，直接告诉前端当前用户角色）
  → 前端：token 存 localStorage，role 存全局状态
  → 跳转到对应首页
```

### 2.2 请求拦截器（自动带 Token）

```javascript
axios.interceptors.request.use(config => {
  const token = localStorage.getItem('token');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});
```

### 2.3 响应拦截器（统一错误处理）

```javascript
// 401 = Token 过期/无效 → 跳转登录
// 403 = 权限不足 → 提示无权限
axios.interceptors.response.use(
  response => response,
  error => {
    if (error.response?.status === 401) {
      localStorage.removeItem('token');
      navigate('/login');
    }
    if (error.response?.status === 403) {
      alert('无权限访问');
    }
    return Promise.reject(error);
  }
);
```

### 2.4 登出

```javascript
// JWT 无状态，前端删除 token 即完成登出
localStorage.removeItem('token');
navigate('/login');
```

## 3. 路由守卫

### 3.1 两层守卫分工

| 层 | 做什么 | 防谁 |
|---|--------|------|
| **前端路由守卫** | 根据角色隐藏/显示页面和菜单 | 防普通用户**看到**不该看的界面（用户体验） |
| **后端 rbac** | 根据 JWT 签名中的角色拒绝/放行请求 | 防普通用户**调用**不该调的接口（真正的安全） |

前端信任后端返回的角色来决定 UI 展示，后端绝不信任前端传来的任何东西，只认 JWT 签名。

### 3.2 AuthGuard（登录守卫）

```javascript
function AuthGuard({ children }) {
  const token = localStorage.getItem('token');
  if (!token) return <Navigate to="/login" />;
  return children;
}
```

### 3.3 RoleGuard（角色守卫）

```javascript
function RoleGuard({ roles, children }) {
  const { role } = useAuth();
  if (!roles.includes(role)) return <Navigate to="/" />;
  return children;
}
```

### 3.4 菜单权限（根据角色动态渲染）

```javascript
const menuConfig = [
  { key: 'dashboard', label: '首页' },
  { key: 'inbound', label: '采购入库' },
  { key: 'outbound', label: '销售出库' },
  { key: 'return', label: '退货管理' },
  { key: 'inventory', label: '库存查询' },
  { key: 'transactions', label: '流水查询' },
  { key: 'products', label: '商品管理', roles: ['ADMIN'] },
  { key: 'warehouses', label: '仓库管理', roles: ['ADMIN'] },

const visibleMenu = menuConfig.filter(
  item => !item.roles || item.roles.includes(userRole)
);
```

## 4. 前端实现难点

### 4.1 表格编辑的 state 管理

**问题**：如果用一个大 state 存整张表数据，提交时全量发送会导致后端做大量无意义更新。

**方案一：只记录改过的行**

```javascript
const [data, setData] = useState([]);              // 表格展示数据
const [changedRows, setChangedRows] = useState({}); // { id: { 改了的字段 } }

const handleChange = (id, field, value) => {
  setData(prev => prev.map(item =>
    item.id === id ? { ...item, [field]: value } : item
  ));
  setChangedRows(prev => ({
    ...prev,
    [id]: { ...prev[id], [field]: value }  // 只记改了的字段
  }));
};

// 提交时只发改过的行
await axios.put('/api/products/batch', { updates: changedRows });
```

**方案二（推荐）：单行编辑单行提交**

不在表格里行内编辑，点"编辑"弹表单，改完直接 `PUT /api/products/:sku` 提交单行。对 WMS 后台完全够用，不需要批量编辑。

### 4.2 分页

后端返回 `skip`/`take` 分页数据，前端传 `page` 和 `pageSize`：

```javascript
// 前端请求
const page = 1, pageSize = 20;
const res = await axios.get(`/api/products?page=${page}&pageSize=${pageSize}`);
// 响应: { list: [...], total: 150, page: 1, pageSize: 20 }
```

```javascript
// 后端实现：查询语句不变，只在末尾加 skip/take
const { page = 1, pageSize = 20 } = req.query;
const skip = (page - 1) * pageSize;

const [list, total] = await prisma.$transaction([
  prisma.product.findMany({ skip, take: pageSize, where }),  // where 条件不变
  prisma.product.count({ where }),
]);
```

### 4.3 入库/出库表单 + 下拉框联动

```javascript
// 页面加载时拉取商品和仓库列表
const [products, setProducts] = useState([]);
const [warehouses, setWarehouses] = useState([]);

useEffect(() => {
  axios.get('/api/products').then(res => setProducts(res.data.data));
  axios.get('/api/warehouses').then(res => setWarehouses(res.data.data));
}, []);

// 表单中用 select 下拉框
<select onChange={e => setSelectedProduct(e.target.value)}>
  {products.map(p => <option key={p.sku} value={p.sku}>{p.sku} - {p.name}</option>)}
</select>

<select onChange={e => setSelectedWarehouse(e.target.value)}>
  {warehouses.map(w => <option key={w.code} value={w.code}>{w.code} - {w.name}</option>)}
</select>
```

扫码枪支持：监听 input 的 Enter 键，自动聚焦到下一个输入框。

### 4.4 JWT 安全要点

| 要点 | 说明 |
|------|------|
| Token 存哪 | localStorage |
| Token 有效期 | 后端设置 `expiresIn`（如 2h），过期返回 401，前端跳转登录 |
| Payload 可读 | base64 明文，不要放密码等敏感信息 |
| 不可伪造 | 攻击者能读 payload 但算不出签名 |
| 登出 = 删 token | 前端删除 localStorage 中的 token 即失效 |
