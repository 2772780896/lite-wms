# WMS Lite API 接口文档

> 所有接口统一前缀 `/api`，响应格式统一为 `{ code, data, message }`。
> 需登录接口在 Header 中携带 `Authorization: Bearer <token>`。

---

## 认证

| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| POST | `/api/auth/login` | Public | 登录 |

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名 |
| password | string | 是 | 密码（明文传输，生产环境需 HTTPS） |

**响应：**

| 字段 | 说明 |
|------|------|
| data.token | JWT token，后续请求放在 Authorization Header |
| data.role | 用户角色（`ADMIN` / `STAFF`），明文返回供前端控制 UI |

**错误码：**

| code | 说明 |
|------|------|
| 400 | 参数缺失 |
| 401 | 用户名或密码错误 |

## 商品管理

| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| POST | `/api/products` | ADMIN | 新增商品 |
| PUT | `/api/products/:id` | ADMIN | 编辑商品 |
| GET | `/api/products` | ALL | 商品列表（支持模糊搜索 + 分页） |
| GET | `/api/products/:id` | ALL | 商品详情（含各仓库库存） |

> ALL 可查：仓库人员入库/出库时需要通过下拉框选择商品。

**请求参数：**

| 接口 | 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|------|
| POST /products | sku | string | 是 | 商品编码（唯一） |
| | name | string | 是 | 商品名称 |
| | spec | string | 否 | 规格描述 |
| PUT /products/:id | name | string | 否 | 商品名称 |
| | spec | string | 否 | 规格描述 |
| GET /products | keyword | string | 否 | 模糊搜索（匹配 name 或 sku） |
| | page | number | 否 | 页码，默认 1 |
| | pageSize | number | 否 | 每页条数，默认 20 |

**响应示例：**

POST /products、PUT /products/:id 返回 `data`：

```json
{
  "id": 1,
  "sku": "SKU-001",
  "name": "iPhone 15",
  "spec": "黑色 256G",
  "createdAt": "2026-06-20T10:00:00.000Z",
  "updatedAt": "2026-06-20T10:00:00.000Z"
}
```

GET /products 返回 `data`：

```json
{
  "list": [/* Product 对象数组 */],
  "total": 56,
  "page": 1,
  "pageSize": 20
}
```

GET /products/:id 返回 `data`（含各仓库库存）：

```json
{
  "id": 1,
  "sku": "SKU-001",
  "name": "iPhone 15",
  "spec": "黑色 256G",
  "createdAt": "...",
  "updatedAt": "...",
  "inventory": [
    {
      "id": 1,
      "productId": 1,
      "warehouseId": 1,
      "quantity": 100,
      "warehouse": {
        "id": 1,
        "code": "WH-001",
        "name": "主仓库",
        "address": "上海市..."
      }
    }
  ]
}
```

**错误码：**

| 接口 | code | 说明 |
|------|------|------|
| POST /products | 400 | 参数缺失 或 sku 已存在 |
| PUT /products/:id | 404 | 商品不存在 |
| GET /products/:id | 404 | 商品不存在 |

## 仓库管理

| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| POST | `/api/warehouses` | ADMIN | 新增仓库 |
| PUT | `/api/warehouses/:id` | ADMIN | 编辑仓库 |
| GET | `/api/warehouses` | ALL | 仓库列表 |
| GET | `/api/warehouses/:id` | ALL | 仓库详情 |

> ALL 可查：仓库人员入库/出库时需要通过下拉框选择仓库。

---

## 入库 / 出库 / 退货

| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| POST | `/api/inbound` | ALL | 采购入库（SKU + 仓库编码 + 数量） |
| POST | `/api/outbound` | ALL | 销售出库（SKU + 仓库编码 + 数量） |
| POST | `/api/return` | ALL | 退货（type: RETURN_IN / RETURN_OUT，remark 必填） |

> 需求原文："仓库人员仅可进行入库、出库、库存查询操作"，因此对所有角色开放。

---

## 库存查询 / 流水查询

| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| GET | `/api/inventory` | ALL | 库存查询（按 productId 或 warehouseId 筛选） |
| GET | `/api/inventory/summary` | ALL | 库存汇总（按商品跨仓库求和） |
| GET | `/api/transactions` | ALL | 流水查询（按商品/仓库/时间/类型筛选） |
