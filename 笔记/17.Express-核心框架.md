# Express 核心概念手册

> Express 是在 Node.js http 模块上封装的 Web 工具包。路由、参数解析、中间件这些在原生 Node 里需要手写，Express 帮你标准化了。

---

## 一、中间件（Middleware）

中间件是 Express 的核心。它就是一个函数 `(req, res, next)`，在请求到达最终路由之前先经过它。

### 1.1 执行顺序

所有中间件按 `app.use()` 的**注册顺序**执行，每个必须调 `next()` 才跳到下一个：

```
请求进来 → cors() → express.json() → router → 404兜底 → 错误处理
```

| 调用方式 | 作用 | 下一个执行 |
|----------|------|-----------|
| `next()` | 正常传递 | 普通中间件 |
| `next(err)` | 传递错误 | 跳到错误处理中间件 |
| `res.json()` | 返回响应 | **请求结束**，不再执行 |

### 1.2 内置与第三方中间件

```javascript
app.use(cors())            // 处理跨域
app.use(express.json())    // 解析 JSON 请求体到 req.body
app.use(express.static('public'))  // 托管静态文件
```

`express.json()` 做了什么：检查 Content-Type → 流式读取请求体 → `JSON.parse` → 结果挂到 `req.body`。没有它，`req.body` 就是 `undefined`。

### 1.3 自定义中间件

```javascript
app.use((req, res, next) => {
  console.log(`${req.method} ${req.url}`)
  next()  // 必须调，否则请求挂起
})
```

### 1.4 两种中间件模式

**普通中间件** — 不需要参数，直接是一个函数：

```javascript
function auth(req, res, next) { ... }
module.exports = auth

// 使用：直接传函数
router.post('/products', auth, handler)
```

**工厂函数中间件** — 需要参数，函数返回函数（闭包）：

```javascript
function rbac(allowedRoles) {         // 路由定义时调用，allowedRoles 被闭包记住
  return (req, res, next) => {        // 路由执行时 Express 传入 req/res/next
    if (!allowedRoles.includes(req.user.role)) {
      return res.status(403).json({ message: '权限不足' })
    }
    next()
  }
}
module.exports = rbac

// 使用：先调用工厂函数，返回的函数才是中间件
router.post('/products', auth, rbac(['ADMIN']), handler)
//                             ↑ 这里调用了 rbac(['ADMIN'])，返回中间件函数
```

**区别**：`auth` 不需要参数 → 直接是中间件；`rbac` 需要参数 → 工厂函数。

### 1.5 404 兜底 vs 错误处理

```javascript
// 路由（正常逻辑调 res.json 后请求结束）
app.use('/api', routes)

// 404 兜底（3 个参数）：路由没匹配到时才走到这
// 因为没人调过 res.json()，请求还没结束，所以能走到这里
app.use((req, res) => {
  res.status(404).json({ message: '接口不存在' })
})

// 错误处理（4 个参数）：只有 throw / next(err) 时才触发
app.use((err, req, res, next) => {
  res.status(err.status || 500).json({ message: err.message })
})
```

| 场景 | 谁响应 | 走 404？ | 走错误处理？ |
|------|--------|---------|------------|
| 路由匹配成功 → `res.json()` | 路由自己 | 否 | 否 |
| 路由没匹配到 | 404 兜底 | 是 | 否 |
| 路由中 `next(err)` | 错误处理 | 否 | 是 |

## 二、路由（Router）

### 2.1 Router 也是中间件

`express.Router()` 是一个**迷你 Express 应用**，本质也是一个中间件：

```javascript
const router = express.Router()     // 创建迷你应用
router.post('/products', handler)   // 在迷你应用里定义路由
app.use('/api', router)             // 挂载到主应用（路径拼接）
```

```
请求 POST /api/products
  → app.use('/api', router)   // 匹配到 /api 前缀
    → router 内部匹配 /products
      → 执行 handler
```

Router 没匹配到 → `next()` → 继续往下走（所以才会走到 404 兜底）。

### 2.2 路径拼接

```javascript
// routes/index.js（相对路径）
router.get('/products', handler)     // 相对路径：/products
router.post('/auth/login', handler)  // 相对路径：/auth/login

// app.js（挂载时拼接前缀）
app.use('/api', routes)

// 实际路径：/api/products、/api/auth/login
```

### 2.3 路由中间件链

```javascript
router.post(path, middleware1, middleware2, handler)
```

Express 按顺序执行：middleware1 → middleware2 → handler。每个调 `next()` 跳到下一个。

```javascript
// 先 auth 验签 → 再 rbac 验角色 → 最后执行业务逻辑
router.post('/products', auth, rbac(['ADMIN']), (req, res) => {
  res.json({ code: 0, data: '...' })
})
```

### 2.4 模块化组织

**单文件方式**（当前 WMS 项目）：所有路由集中在一个 `routes/index.js`。

```
backend/
├── app.js                  # 入口：中间件配置 + 路由挂载 + 错误处理
├── routes/
│   └── index.js            # 所有路由定义
├── middleware/
│   ├── auth.js             # 认证中间件
│   └── rbac.js             # 角色守卫中间件
```

**多文件批量注册**（路由增多时拆分）：

```javascript
// routes/index.js — 路由索引（批量注册）
const authRoutes = require('./authRoutes')
const videoRoutes = require('./videoRoutes')

// 导出路由配置数组
module.exports = [
  { path: '/api/auth', router: authRoutes },
  { path: '/api/video', router: videoRoutes },
]
```

```javascript
// app.js — 批量挂载
const routes = require('./routes')

routes.forEach(({ path, router }) => {
  app.use(path, router)
})
```

| 方式 | 适用场景 |
|------|----------|
| 单文件 `routes/index.js` | 路由较少（< 30 条），项目初期 |
| 多文件 + 批量注册 | 路由较多，需要按业务模块拆分 |

## 三、请求与响应

### 3.1 req 常用属性

| 属性 | 说明 | 示例 |
|------|------|------|
| `req.query` | URL 查询参数（`?key=val`） | `{ page: '1' }` |
| `req.params` | 路径参数（`:id`） | `{ id: '123' }` |
| `req.body` | 请求体（需 `express.json()`） | `{ name: 'test' }` |
| `req.headers` | 请求头（键名小写） | `{ authorization: 'Bearer ...' }` |
| `req.method` | HTTP 方法 | `'POST'` |

### 3.2 res 常用方法

| 方法 | 说明 |
|------|------|
| `res.json(data)` | 返回 JSON（自动设 Content-Type） |
| `res.status(code)` | 设置状态码（链式调用） |
| `res.send(data)` | 返回任意类型 |
| `res.redirect(path)` | 重定向 |

链式调用：`res.status(201).json({ code: 201, data: user })`

### 3.3 状态码语义

| 码 | 含义 | 场景 |
|----|------|------|
| 200 | OK | 查询成功 |
| 201 | Created | 创建成功 |
| 400 | Bad Request | 参数有误 |
| 401 | Unauthorized | 未登录 / token 无效 |
| 403 | Forbidden | 权限不足 |
| 404 | Not Found | 资源不存在 |
| 500 | Internal Error | 服务器错误 |

## 四、错误处理

### 4.1 错误从哪来

| 来源 | err 格式 |
|------|----------|
| 主动抛出 | `new Error('xxx')`，可自定义 `err.status` |
| Prisma | `PrismaClientKnownRequestError`，`err.code` 如 P2002（唯一约束） |
| JWT | `err.name` 为 `JsonWebTokenError` 或 `TokenExpiredError` |

### 4.2 错误传递链

```javascript
// 路由中：try/catch 捕获 → next(err) 传给全局处理
router.post('/products', async (req, res, next) => {
  try {
    await prisma.product.create({ data: req.body })
    res.json({ code: 0 })
  } catch (err) {
    next(err)  // → 跳到错误处理中间件
  }
})

// 全局错误处理：根据错误类型返回不同状态码
app.use((err, req, res, next) => {
  let status = err.status || 500
  let message = err.message || '服务器内部错误'

  // Prisma 唯一约束
  if (err.code === 'P2002') {
    status = 400
    message = '数据已存在'
  }

  res.status(status).json({ code: status, data: null, message })
})
```

### 4.3 自定义错误类

```javascript
class AppError extends Error {
  constructor(message, status = 500) {
    super(message)
    this.status = status
  }
}

// 使用
throw new AppError('用户名已存在', 400)
```

## 五、标准 app.js 模板

```javascript
require('dotenv').config()
const express = require('express')
const cors = require('cors')
const routes = require('./routes')

const app = express()

// 1. 全局中间件
app.use(cors())
app.use(express.json())

// 2. 路由挂载（Router 也是中间件）
app.use('/api', routes)

// 3. 404 兜底（路由没匹配到时）
app.use((req, res) => {
  res.status(404).json({ code: 404, data: null, message: '接口不存在' })
})

// 4. 全局错误处理（4 参数签名，只有 next(err) 时触发）
app.use((err, req, res, next) => {
  console.error(err.message)
  res.status(err.status || 500).json({
    code: err.status || 500, data: null, message: err.message
  })
})

// 5. 启动
const PORT = process.env.PORT || 3000
app.listen(PORT, () => console.log(`Server: http://localhost:${PORT}`))
```
