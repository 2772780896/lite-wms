# Node.js 基础手册

> Node.js 运行时核心：模块化、内置模块、npm 包管理。
> Express、Prisma、JWT 等进阶内容已拆分至独立手册。

---

## 一、运行环境

- **Node.js**：JS 的后端运行环境，脱离浏览器运行。
- **限制**：无法使用 DOM 和 BOM。
- **运行**：终端输入 `node xxx.js`。

---

## 二、导入导出（模块化）

### CommonJS (CJS) - Node 默认

- **导出**：`module.exports = { name: 'zs' }`
- **导入**：`const m = require('./xxx.js')`

### ECMAScript (ESM) - 现代标准

- **配置**：`package.json` 写入 `{"type": "module"}`。
- **导出**：`export default { ... }` 或 `export const a = 1`。
- **导入**：`import m from './xxx.js'` 或 `import { a } from './xxx.js'`。

## 三、Map：键值对数据结构

类似对象 `{}`，但更纯粹的键值对存储。

```javascript
const tasks = new Map()

tasks.set('abc', { status: 'done' })   // 存入
tasks.get('abc')                        // 取出 → { status: 'done' }
tasks.has('abc')                        // 是否存在 → true
tasks.delete('abc')                     // 删除
tasks.size                              // 数量
```

**为什么用 Map 而不是对象**：
- 没有原型链干扰（对象会继承 `toString`、`constructor` 等属性）
- `get/set/has` 语义更清晰
- key 可以是任意类型（对象、函数都行）

**局限**：数据存在内存里，服务器重启就丢了。持久化需换数据库。

**典型场景**：临时全局状态，如任务队列、在线用户列表、缓存。

## 四、核心内置模块

### 1. path：路径处理

| 方法 | 作用 | 特点 |
|------|------|------|
| `path.join(...paths)` | 拼接路径 | 原样拼接，`../` 保留 |
| `path.resolve(...paths)` | 解析绝对路径 | 像 `cd` 一样走，`../` 会消掉 |
| `__dirname` | 当前文件的父目录绝对路径 | 不需要引入 |

```javascript
const path = require('path')

// __dirname = "/project/backend"
path.resolve(__dirname, '../.env')   // → "/project/.env"
path.join(__dirname, '../.env')      // → "/project/backend/../.env"
```

### 2. fs：文件读写与目录操作

**异步写法**（建议用 `fs.promises` 以支持 `async/await`）：

```javascript
const fs = require('fs').promises

await fs.writeFile('./test.txt', '内容', 'utf-8')
const data = await fs.readFile('./test.txt', 'utf-8')
// 不加 'utf-8' 返回 Buffer，需 data.toString() 转换
```

**目录操作**（同步，常用于启动时初始化）：

```javascript
const fs = require('fs')

if (!fs.existsSync('./uploads')) {
  fs.mkdirSync('./uploads', { recursive: true })  // recursive: true 递归创建
}
```

**典型用法：启动时确保目录存在**：

```javascript
const dirs = ['input', 'temp', 'output']
dirs.forEach(dir => {
  if (!fs.existsSync(path.join(__dirname, dir))) {
    fs.mkdirSync(path.join(__dirname, dir), { recursive: true })
  }
})
```

### 3. http：创建 Web 服务（了解原理）

```javascript
const http = require('http')

const server = http.createServer((req, res) => {
  res.setHeader('Content-Type', 'text/html; charset=utf-8')
  if (req.url === '/index') {
    res.end('首页')
  }
})
server.listen(3000, () => console.log('启动在 3000'))
```

> Express 就是在这个基础上封装的工具包。

## 五、npm 包管理

### 基础命令

| 命令 | 作用 |
|------|------|
| `npm init -y` | 初始化 `package.json` |
| `npm i 包名` | 安装包，记录到 `dependencies` |
| `npm i` | 根据清单重装所有包 |
| `npm uni 包名` | 删除包 |

### nodemon

- **作用**：代码保存后自动重启服务。
- **安装**：`npm i -g nodemon`。

### 标准项目结构

```
project/
├── app.js                  # 入口文件
├── package.json
├── config/                 # 配置（数据库连接、端口、密钥）
├── routes/                 # 路由层（请求分发）
├── controllers/            # 控制器层（处理请求、返回响应）
├── middlewares/             # 中间件（通用拦截）
├── lib/                    # 基础设施（数据库连接实例等）
└── utils/                  # 工具函数
```
