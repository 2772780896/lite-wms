# Prisma ORM 手册

> Prisma 是 Node.js 的 ORM（对象关系映射），运行在 Node.js 环境中，不依赖 Express。
> 它通过 `schema.prisma` 定义数据模型，自动生成类型安全的数据库操作 API。

---

## 一、快速上手

### 1. 安装与初始化

```bash
npm install prisma @prisma/client --save-dev  # prisma 是 CLI，@prisma/client 是运行时
npx prisma init                               # 生成 schema.prisma + .env
```

### 2. 定义模型（schema.prisma）

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "sqlite"        // 学习用 SQLite，生产用 mysql/postgresql
  url      = env("DATABASE_URL")
}

model User {
  id       Int      @id @default(autoincrement())
  username String   @unique
  password String
  role     String   @default("STAFF")
  createdAt DateTime @default(now())
  posts    Post[]   // 反向关系：一个用户有多篇文章
}

model Post {
  id        Int      @id @default(autoincrement())
  title     String
  authorId  Int                                          // 外键字段（真实存储）
  author    User     @relation(fields: [authorId], references: [id])  // 虚拟字段（关联关系）
  createdAt DateTime @default(now())
}
```

### 3. 同步数据库

```bash
npx prisma db push     # 将 schema 推送到数据库（开发用）
npx prisma generate    # 生成 Prisma Client（npm install 后需重新执行）
```

### 4. 创建连接实例

```javascript
// lib/prisma.js — 全局单例，所有控制器共享
const { PrismaClient } = require('@prisma/client')
const prisma = new PrismaClient()
module.exports = prisma
```

> **为什么全局单例**：每个 `new PrismaClient()` 创建一个连接池，多个实例会浪费资源。

## 二、CRUD 操作

### 1. 增（Create）

```javascript
const user = await prisma.user.create({
  data: { username: 'alice', password: 'hashed_pw', role: 'ADMIN' }
})
// 返回值：{ id: 1, username: 'alice', password: 'hashed_pw', role: 'ADMIN', createdAt: '...' }
```

### 2. 删（Delete）

```javascript
await prisma.user.delete({ where: { id: 1 } })
// 返回值：被删除的那条记录
```

### 3. 改（Update）

```javascript
const user = await prisma.user.update({
  where: { id: 1 },
  data: { role: 'ADMIN' }
})
// 返回值：更新后的完整记录
```

### 4. 查（Find）

```javascript
// 查单条（按主键或唯一字段）
const user = await prisma.user.findUnique({ where: { id: 1 } })
// 找不到返回 null

// 查多条
const users = await prisma.user.findMany()

// 条件查询（模糊匹配）
const result = await prisma.user.findMany({
  where: { username: { contains: 'ali' } }
})

// 排序 + 分页
const result = await prisma.user.findMany({
  skip: 0,
  take: 10,
  orderBy: { createdAt: 'desc' }
})
```

### 5. Upsert（有则更新，无则创建）

```javascript
const inventory = await prisma.inventory.upsert({
  where: { productId_warehouseId: { productId: 1, warehouseId: 1 } },  // 需 @@unique 联合约束
  update: { quantity: { increment: 10 } },
  create: { productId: 1, warehouseId: 1, quantity: 10 }
})
```

> 前提：schema 中必须有 `@@unique([productId, warehouseId])` 联合唯一约束。

## 三、返回值速查

| 操作 | 返回值 | 找不到时 |
|------|--------|---------|
| `create` | 创建的记录对象 | — |
| `findUnique` | 记录对象 | `null` |
| `findMany` | 记录数组 `[]` | `[]`（空数组） |
| `update` | 更新后的记录 | 抛 `P2025` |
| `upsert` | 创建或更新后的记录 | — |
| `delete` | 被删除的记录 | 抛 `P2025` |
| `updateMany` | `{ count: 影响行数 }` | `{ count: 0 }` |
| `count` | 数字 | `0` |
| `$transaction([...])` | 数组 `[结果1, 结果2, ...]` | — |

**注意**：`updateMany` 不返回具体记录，只返回影响行数。这就是为什么用它做"乐观锁"扣减时，通过 `result.count === 0` 判断是否失败。

## 四、select 与 in

### select：只查你要的字段

```javascript
// 默认返回全部字段
const users = await prisma.user.findMany()
// → [{ id: 1, username: 'alice', password: '...', role: 'ADMIN', createdAt: '...' }]

// 用 select 只取指定字段
const users = await prisma.user.findMany({
  select: { id: true, username: true }
})
// → [{ id: 1, username: 'alice' }]
```

### in：批量匹配一组值

```javascript
const ids = [1, 2, 3]
const users = await prisma.user.findMany({
  where: { id: { in: ids } }   // 等价于 SQL: WHERE id IN (1, 2, 3)
})
```

### 实战：批量查交互状态

```javascript
// 1. 拿到当前页所有 ID
const postIds = [5, 8, 12, 17]

// 2. 一次查出"当前用户点赞过哪些"（1 次查询 vs 循环查 4 次）
const likes = await prisma.userLike.findMany({
  where: { uid: currentUid, item_id: { in: postIds } },
  select: { item_id: true }
})
// → [{ item_id: 5 }, { item_id: 12 }]

// 3. 用 Set 快速判断
const likeSet = new Set(likes.map(l => l.item_id))
// likeSet.has(5) → true, likeSet.has(8) → false
```

| 写法 | 查询次数 |
|------|---------|
| 循环查（每条一次） | N 次 |
| `in` 批量查 | **1 次** |

## 五、@relation 与 include

### 核心概念

- `@relation`：在 schema 里**声明关系**（只写一次）
- `include`：在查询时**触发连表**（每次查询可选）

### 不用 include：各查各的

```javascript
const post = await prisma.post.findUnique({ where: { id: 1 } })
// → { id: 1, title: '...', authorId: 42 }  // 只有外键数字，没有作者信息

const author = await prisma.user.findUnique({ where: { id: post.authorId } })
// 手动再查一次
```

### @relation + include：自动连表

```javascript
const post = await prisma.post.findUnique({
  where: { id: 1 },
  include: { author: true }   // 帮我把作者信息也查出来
})
// → { id: 1, title: '...', authorId: 42, author: { id: 42, username: 'alice', ... } }
```

**底层过程**：
```
1. 查 Post 表 WHERE id = 1 → 得到 authorId = 42
2. 翻 @relation 说明书：用 authorId 去 User 表找 id
3. 查 User 表 WHERE id = 42 → 把结果塞进 author 字段
```

### 嵌套 include：多层连表

```javascript
const post = await prisma.post.findUnique({
  where: { id: 1 },
  include: {
    author: true,
    comments: {
      include: { author: true }  // 评论也要带评论者信息
    }
  }
})
```

### include vs select 对比

| | include | select |
|--|---------|--------|
| 作用 | 把关联表数据带进来 | 只取当前表的部分字段 |
| 返回 | 全字段 + 关联数据 | 只返回选中的字段 |
| 场景 | 需要关联信息 | 只需要 ID 做判断 |

## 六、@relation 命名关系

### 何时需要命名

| 情况 | 要命名吗 | 例子 |
|------|---------|------|
| A 表只有**一个字段**指向 B 表 | 不用 | Post → User（只有 `authorId`） |
| A 表有**两个字段**都指向 B 表 | 要 | Dialogue → User（`participant1Id` + `participant2Id`） |

### 命名关系写法

```prisma
model Dialogue {
  participant1Id  Int
  participant2Id  Int
  participant1    User  @relation("P1", fields: [participant1Id], references: [id])
  participant2    User  @relation("P2", fields: [participant2Id], references: [id])
}

model User {
  dialogues1  Dialogue[]  @relation("P1")
  dialogues2  Dialogue[]  @relation("P2")
}
```

## 七、错误码速查

| 错误码 | 含义 | 场景 |
|--------|------|------|
| `P2002` | 唯一约束冲突 | create 时字段值重复 |
| `P2025` | 记录不存在 | update/delete 时找不到目标 |

```javascript
try {
  await prisma.product.create({ data: { sku: 'SKU-001', name: '...' } })
} catch (err) {
  if (err.code === 'P2002') {
    // sku 已存在
    return res.status(400).json({ message: '商品编码已存在' })
  }
  next(err)  // 其他错误交给全局错误处理
}
```

## 八、事务

### 交互式事务（多步操作需要回滚）

```javascript
await prisma.$transaction(async (tx) => {
  // 步骤 1：更新库存
  const result = await tx.inventory.updateMany({
    where: { productId, warehouseId, quantity: { gte: qty } },
    data: { quantity: { decrement: qty } }
  })
  if (result.count === 0) throw new Error('库存不足')

  // 步骤 2：写入流水（如果失败，步骤 1 自动回滚）
  await tx.transactionLog.create({
    data: { productId, warehouseId, type: 'OUTBOUND', quantity: qty, operatorId }
  })
})
```

### 批量事务（多条独立查询一起发）

```javascript
const [list, total] = await prisma.$transaction([
  prisma.product.findMany({ skip, take: pageSize }),  // 查列表
  prisma.product.count(),                               // 查总数
])
// 两条 SQL 一起发送，减少网络往返
```

## 九、对比原生 SQL

| | 原生 SQL | Prisma |
|--|---------|--------|
| 关系定义 | 每次查询时写在 SQL 里 | 在 schema 里写一次 |
| 查询写法 | `LEFT JOIN User ON p.authorId = u.id` | `include: { author: true }` |
| 返回结构 | 扁平一行，要自己组装嵌套 | 自动嵌套好 |
| 类型提示 | 无（运行时才报错） | 有（编译时提示） |
| 灵活性 | 极高 | 受限于 schema 声明的关系 |
