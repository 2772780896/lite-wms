# Express 常用中间件与实战规范

> 中间件详细用法 + RESTful API 设计 + 身份认证实战。

## 1. express.json()

**作用**：解析请求体中的 JSON 数据，结果放在 `req.body` 中。

**没有它会怎样**：`req.body` 为 `undefined`，无法获取前端发送的 JSON 数据。

**底层原理**（简化版）：

```javascript
function jsonParser(req, res, next) {
  if (req.headers['content-type'] !== 'application/json') {
    return next()  // 不是 JSON，跳过
  }
  let body = ''
  req.on('data', chunk => { body += chunk.toString() })
  req.on('end', () => {
    try {
      req.body = JSON.parse(body)
      next()
    } catch (err) {
      res.status(400).json({ error: 'Invalid JSON' })
    }
  })
}
```

## 2. express.static()

**作用**：把指定文件夹变成可通过 URL 访问的静态文件服务。

**语法**：
```javascript
app.use('/url前缀', express.static('文件夹路径'))
```

**示例**：
```javascript
app.use('/files', express.static(path.join(__dirname, 'output')))
// output/abc.mp4 → http://localhost:3000/files/abc.mp4
```

**执行流程**：
```
请求 GET /files/abc.mp4
  → 去掉前缀 /files → 得到 abc.mp4
  → 去 output/ 目录找 abc.mp4
  → 存在 → 自动设 Content-Type → 返回文件流
  → 不存在 → 放行，继续找下一个路由
```

**自动设置的 Content-Type**：

| 后缀 | Content-Type | 浏览器行为 |
|------|-------------|-----------|
| `.mp4` | `video/mp4` | 可直接播放 |
| `.png/.jpg` | `image/png` / `image/jpeg` | 显示图片 |
| `.html` | `text/html` | 渲染网页 |
| `.srt` | `application/octet-stream` | 触发下载 |

**缺点**：无权限控制、无自定义错误、无日志。

**对比手动返回文件**：

```javascript
// 路由里手动返回（可做权限校验）
router.get('/download/:taskId/video', (req, res) => {
  const task = tasks.get(req.params.taskId)
  if (!task || task.status !== 'done') {
    return res.status(400).json({ error: '任务未完成' })
  }
  res.sendFile(filePath)   // 发送文件
  // 或 res.download(filePath)  // 强制下载
})
```

| 方式 | 权限控制 | 自定义错误 | 适用场景 |
|------|---------|-----------|---------|
| `express.static` | ❌ | ❌ | 公开资源托管 |
| `res.sendFile()` | ✅ | ✅ | 需要校验的文件访问 |
| `res.download()` | ✅ | ✅ | 强制下载（不预览） |

## 3. multer（文件上传）

**安装**：`npm i multer`

**作用**：解析 `multipart/form-data` 请求体，将文件保存到磁盘，文件信息写入 `req.file`。

**基本用法**：

```javascript
const multer = require('multer')
const upload = multer({ dest: 'uploads/' })

router.post('/upload', upload.single('video'), (req, res) => {
  // req.file = 上传的文件信息
  // req.body = 其他表单字段
})
```

`upload.single('fieldName')` 是一个**中间件函数**，`'video'` 是前端 FormData 中的字段名。

**前端对应**：
```javascript
const formData = new FormData()
formData.append('video', fileObject)   // 必须和 upload.single('video') 一致
formData.append('style', jsonStr)      // 普通字段，放在 req.body 里
```

**req.file 属性**：

| 属性 | 说明 | 示例 |
|------|------|------|
| `fieldname` | 表单字段名 | `'video'` |
| `originalname` | 原始文件名 | `'我的视频.mp4'` |
| `filename` | 保存后的文件名 | `'abc123.mp4'` |
| `path` | 保存路径 | `'uploads/abc123.mp4'` |
| `size` | 文件大小（字节） | `5242880` |
| `mimetype` | MIME 类型 | `'video/mp4'` |

**自定义存储**：

```javascript
const storage = multer.diskStorage({
  destination: (req, file, cb) => {
    cb(null, path.join(__dirname, '../input'))
  },
  filename: (req, file, cb) => {
    const ext = path.extname(file.originalname)
    cb(null, `${uuidv4()}${ext}`)  // UUID 重命名，防重名
  }
})
const upload = multer({ storage })
```

**cb 格式**：`cb(错误, 结果)`，第一个传 `null` 表示无错误。

**多文件上传**：

| 方法 | 说明 |
|------|------|
| `upload.single('field')` | 单个文件 |
| `upload.array('field', maxCount)` | 多个同名文件 |
| `upload.fields([{ name: 'a' }, { name: 'b' }])` | 多个不同字段 |

## 4. dotenv

**作用**：读取 `.env` 文件，注入到 `process.env`。

```env
# .env
DATABASE_URL="file:./dev.db"
JWT_SECRET="my_secret_key_123"
```

```javascript
require('dotenv').config()
// process.env.DATABASE_URL → "file:./dev.db"
```

> Prisma 自带 `.env` 读取（CLI 命令时自动加载），`dotenv` 是给 `app.js` 运行时用的。

## 5. cors

**作用**：处理跨域请求。

```javascript
app.use(cors())  // 允许所有来源

// 或配置特定来源
app.use(cors({
  origin: 'http://localhost:5173',  // 只允许前端开发地址
  credentials: true,                 // 允许携带 cookie
}))
```

## 6. RESTful API 设计规范

**核心思想**：URL 代表资源，HTTP Method 代表动作。

### 标准写法

| 动作 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 查列表 | GET | `/users` | 获取用户列表 |
| 查详情 | GET | `/users/:id` | 获取指定用户 |
| 新增 | POST | `/users` | 创建用户（数据在 `req.body`） |
| 修改 | PUT | `/users/:id` | 更新指定用户 |
| 删除 | DELETE | `/users/:id` | 删除指定用户 |

### 状态码规范

| 码 | 含义 | 场景 |
|----|------|------|
| 200 | OK | 查询/更新成功 |
| 201 | Created | 创建成功 |
| 400 | Bad Request | 参数有误 |
| 401 | Unauthorized | 未登录 / token 无效 |
| 403 | Forbidden | 权限不足 |
| 404 | Not Found | 资源不存在 |
| 500 | Internal Error | 服务器错误 |

## 7. 身份认证

### Session 模式（了解原理）

**原理**：用户登录后，服务器在内存存储用户信息，给浏览器发 `connect.sid` Cookie。下次请求带上 Cookie 即可识别。

```javascript
const session = require('express-session')

app.use(session({
  secret: 'my_secret',
  resave: false,
  saveUninitialized: true
}))

// 登录时存
app.post('/login', (req, res) => {
  req.session.user = req.body
  req.session.isLogin = true
  res.send('登录成功')
})

// 验证时取
app.get('/profile', (req, res) => {
  if (!req.session.isLogin) return res.status(401).send('未登录')
  res.send({ username: req.session.user.username })
})
```

**局限**：无法跨域，服务器重启后 Session 丢失。

### JWT 模式（推荐）

**原理**：无状态，token 里包含用户信息（Base64 编码），服务器不存任何东西。

```javascript
const jwt = require('jsonwebtoken')
const SECRET = process.env.JWT_SECRET  // 签名密钥

// 1. 登录成功 → 签发 token
const token = jwt.sign(
  { userId: 1, role: 'ADMIN' },  // payload（用户信息）
  SECRET,                         // 密钥
  { expiresIn: '2h' }            // 过期时间
)
res.json({ token, role: 'ADMIN' })

// 2. 后续请求 → 验证 token
const authMiddleware = (req, res, next) => {
  const token = req.headers.authorization?.split(' ')[1]  // "Bearer xxx" → "xxx"
  if (!token) return res.status(401).json({ message: '未登录' })

  try {
    req.user = jwt.verify(token, SECRET)  // 验签 + 解出 payload
    next()
  } catch (err) {
    res.status(401).json({ message: 'token 无效或已过期' })
  }
}
```

**jwt.verify 自动验证什么**：
- 签名是否被篡改
- token 是否过期（`exp` 字段）
- 都通过 → 返回 payload；任一失败 → 抛错
